## ðŸ“± Mobile Edit Profile Screen
**File Reference:** `mobile/app/edit-profile.tsx`

A comprehensive, multi-step wizard for building a rich user persona.

### Navigation & Control
*   **Stepper:** 6-step process (Basics -> Professional -> Vibe -> Deep Dive -> Favorites -> Stalk Me).
*   **Header:** "Edit Profile" (implied via navigation).
*   **Footer Controls:** "Next" (or "Save" on final step) and "Back" buttons.

### Step 1: The Basics
*   **Avatar:** Tap to upload a profile picture (Circular crop).
*   **Cover Media:** Tap to upload a banner Image or Video.
*   **Fields:**
    *   **Display Name:** Text input.
    *   **Date of Birth:** DD-MM-YYYY format with auto-formatting. Enforces 18+ age restriction.
    *   **Gender:** Selector (Male/Female/Other).
    *   **Height & Weight:** Text inputs.
    *   **Conversation Starter:** Multi-line text area ("The first thing you should ask me is...").

### Step 2: My Professional Side
*   **Work History:** Text area for career background.
*   **Education:** Input for University/High School.
*   **Achievements:** "Proud moments" text area.
*   **Skills:** Tag input system (Type & Add) for listing skills (e.g., "Java", "Leadership").

### Step 3: My Vibe
*   **City:** Location input.
*   **Interaction Mode:** Selector (Introvert / Ambivert / Extrovert).
*   **"With me life is like...":** Descriptive text area for personality vibe.

### Step 4: Deep Dive
*   **Values:** Tag input for core values (e.g., "Growth", "Honesty").
*   **Hard No's:** Red-themed tag input for dealbreakers.
*   **Emotional Fit:** Description of ideal emotional match.

### Step 5: Favorites (Top 3s)
*   **Categories:** Movies, Genres, Songs, Foods, Places, Games.
*   **Input:** Three separate text inputs per category to list top favorites (Ranked #1, #2, #3).

### Step 6: Stalk Me (Media Gallery)
*   **Grid:** 11 Slots for uploading images/videos.
*   **Function:** Allows users to create a visual "shrine" or gallery of their life for others to browse ("Stalk").

### Saving & Logic
*   **Uploads:** Handles media uploads to R2 storage sequentially.
*   **Self-Healing:** Automatically syncs missing public `User` records if a foreign key error occurs during save.
*   **Retry Logic:** Implements exponential backoff for API saves if RLS blocks the direct database write.
*   **Completion:** Updates `profile_completed` flag on success.
